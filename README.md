# Spatio-Temporal Transformer Networks for Hyperlocal Climate Change Prediction

[![Python](https://img.shields.io/badge/Python-3.10+-green)](https://www.python.org/)
[![PyTorch](https://img.shields.io/badge/PyTorch-CUDA-red)](https://pytorch.org/)

Official implementation of the proof-of-concept pilot study from the paper:

> **Spatio-Temporal Transformer Networks for Hyperlocal Climate Change Prediction: A Model Architecture**
> Akshit Naithani — NMIMS University, Mumbai, India

---

## Overview

Global Climate Models (GCMs) operate at coarse spatial resolutions (100–250 km), making them unsuitable for hyperlocal decision-making. This work proposes the **Spatio-Temporal Transformer Network (STTN)**, a hybrid deep learning architecture that combines:

- **ConvLSTM layers** for local spatio-temporal feature extraction
- **Patch-based multi-head self-attention** with learnable positional encodings for long-range dependency modeling
- An **encoder-decoder upsampling head** for spatial resolution enhancement

The pilot study demonstrates **4× spatial downscaling** of ERA5 reanalysis 2m temperature over South Asia, achieving a **44.3% RMSE reduction** over bilinear interpolation and producing physically interpretable attention maps.

---

## Key Results

| Model | RMSE (K) | MAE (K) | Pearson r |
|---|---|---|---|
| Bilinear Interpolation | 3.448 | 2.439 | 0.9624 |
| Vanilla ConvLSTM | 1.885 | 1.253 | 0.9897 |
| **STTN (Ours)** | **1.922** | **1.301** | **0.9894** |

Test period: 2006–2010 | Domain: South Asia (5°–40°N, 60°–100°E) | Variable: 2m Temperature

The spatial attention map assigns higher weights to the Himalayan region without any geographic supervision, demonstrating physically meaningful learned priors.

## Setup

### 1. Clone the repository
```bash
git clone https://github.com/naithaniakshit/sttn-climate.git
cd sttn-climate
```

### 2. Install dependencies
```bash
pip install torch torchvision numpy xarray netcdf4 h5netcdf matplotlib scipy scikit-learn cdsapi
```

### 3. Set up ERA5 API access
Register for a free account at [https://cds.climate.copernicus.eu](https://cds.climate.copernicus.eu) and create a `.cdsapirc` file in your home directory:

```
url: https://cds.climate.copernicus.eu/api/v2
key: <YOUR-UID>:<YOUR-API-KEY>
verify: 1
```

### 4. Run the notebook
Open `STTN_Pilot_Study.ipynb` in Jupyter and run all cells in order. The notebook will:
1. Download ERA5 data (~40 MB)
2. Preprocess and coarsen to simulate GCM input
3. Define and train all three models (bilinear, ConvLSTM, STTN)
4. Evaluate on the held-out test set
5. Generate all figures used in the paper

A CUDA-capable GPU is recommended but not required. On CPU, expect training to take approximately 10–15 minutes per model.

## Model Architecture

```
Input: (B, T=12, H_coarse=35, W_coarse=40)
         │
    ┌────▼────┐
    │ ConvLSTM│  ×2 layers, 32 hidden channels
    │ Encoder │  Captures local spatio-temporal dynamics
    └────┬────┘
         │  Hidden states H_{1:T}  (B, T, 32, 35, 40)
    ┌────▼────────────────┐
    │  Patch Tokenization │  p=4 → (B, T×80, 32)
    │  + Pos. Encoding    │  294× memory reduction vs full attention
    └────┬────────────────┘
         │
    ┌────▼──────────────┐
    │  Multi-Head       │  4 heads, scaled dot-product attention
    │  Self-Attention   │  Models long-range teleconnections
    └────┬──────────────┘
         │  Residual fusion with last ConvLSTM state
    ┌────▼──────────┐
    │  Upsampling   │  TransposedConv ×2 + bilinear resize
    │  Decoder Head │
    └────┬──────────┘
         │
Output: (B, H_fine=141, W_fine=161)
```

## Computational Complexity

A key contribution of this work is a rigorous analysis of attention memory cost at climate-relevant grid sizes:

| Patch size p | Tokens N | Attention Memory | Reduction |
|---|---|---|---|
| 1 (no patch) | 43,200 | 3.51 GB | 1× |
| 2 | 10,800 | 0.22 GB | 16× |
| **4 (recommended)** | **2,520** | **0.014 GB** | **294×** |
| 8 | 504 | 0.001 GB | 7,347× |
| 16 | 72 | <0.001 GB | 360,000× |

Full global ERA5 grid with naive attention would require ~2.3 × 10⁶ GB — entirely infeasible. Patch size p=4 is the recommended operating point, balancing memory efficiency with spatial granularity.
